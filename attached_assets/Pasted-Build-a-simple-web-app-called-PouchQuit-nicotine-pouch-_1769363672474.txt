Build a simple web app called “PouchQuit” (nicotine pouch tracker for quitting Zyn).

Tech:
- Use HTML/CSS/JS only (no frameworks).
- Single-page app.
- Mobile-first UI.
- Save everything in localStorage so it persists after refresh.
- Include clear comments in the code.

Core features:
1) Counter tracker
- Big display: “Pouches today” with +1 and -1 buttons.
- Also show “Total pouches (all time)”.
- Add an optional “log” list: each time user presses +1, store timestamp in a daily log.
- “Undo last” button removes the most recent log entry.

2) Quit challenge / hourly check-in
- On app open, show a modal that asks: “Can you go the next hour without a pouch?”
- Buttons: “Yes” and “No”.
- If Yes: start a 60-minute countdown timer and show it on the home screen.
- If No: show a short, non-judgmental suggestion and let them try again.

3) Money saved tracker
- Settings: cost per can ($), pouches per can, and your usual baseline pouches per day (before quitting).
- Calculate:
  - Estimated pouches avoided = baseline - actual (don’t go below 0).
  - Money saved today, this week, and all-time.
- Show “Cans avoided” estimate too.

4) Advice / coaching (simple but good)
- A “Advice” section that generates tips based on:
  - Time since last pouch
  - How many used today vs baseline
  - Whether they failed the last hourly challenge
- Keep tone practical and encouraging, not cheesy.
- Provide 3 quick actions: “Drink water”, “5 deep breaths”, “Walk 2 minutes”.

5) Streaks & milestones
- Show “Time since last pouch” (in hours/minutes).
- Show “Longest streak”.
- Milestones (badge style): 2 hours, 6 hours, 12 hours, 24 hours, 3 days, 7 days.
- Confetti animation (simple CSS) when a milestone hits.

Pages/Sections (same page, tabs or sections):
- Dashboard (counter, timer, time since last, streaks)
- Money
- Log
- Settings

Settings:
- Wake hours (start/end) to control when the “next hour” prompt shows.
- Baseline pouches/day (default 8).
- Cost/can (default 6.00).
- Pouches/can (default 15).
- Reset data button with confirmation.

UX requirements:
- Big buttons, thumb-friendly.
- Dark mode toggle.
- No external APIs.
- Avoid any medical claims; add a small disclaimer: “Not medical advice”.

Deliverables:
- Create: index.html, styles.css, app.js.
- Make it run immediately in Replit.
